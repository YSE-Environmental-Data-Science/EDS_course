Random Forest Models: Predicting Methane Flux

From integrated data to prediction

In Data Integration, we built the kind of table needed for methane flux modeling: one row per site-month, with methane flux as the response and climate, vegetation, and ecosystem variables as candidate predictors.

In this workshop, we will ask:

Can we use climate, vegetation, and ecosystem predictors to estimate monthly methane flux, and how do we know whether the model is useful?

By the end of this workshop, you will be able to:

  1. Identify a response variable and candidate predictors.
  2. Prepare a model-ready table for random forest regression.
  3. Compare candidate random forest models.
  4. Evaluate train and test performance using RMSE, MAE, R-squared, and bias.
  5. Interpret variable importance without treating it as causation.
  6. Compare random row splits with leave-one-site-out assessment.
  7. Build a final model by selecting the best predictor from each category.
  8. Save a final model object for sensitivity analysis and spatial prediction.

Random forests

Random forests are ensembles of decision trees. Each tree is fit to a bootstrap sample of the data, and each split considers only a random subset of predictors. The final prediction is the average prediction across many trees.

Random forests are useful for environmental data because they can model nonlinear relationships and interactions without requiring us to specify those relationships in advance. They are still models, though. They can overfit, they can inherit bias from the training data, and they cannot reliably predict conditions not represented in the data.

Evaluation metrics

We will evaluate models using four metrics:

  • RMSE: root mean squared error; lower is better.
  • MAE: mean absolute error; lower is better.
  • R-squared: squared correlation between observed and predicted; higher is better.
  • Bias: mean predicted minus observed; closer to zero is better.
model_metrics <- function(observed, predicted) {
  tibble(
    rmse = sqrt(mean((observed - predicted)^2, na.rm = TRUE)),
    mae = mean(abs(observed - predicted), na.rm = TRUE),
    r2 = cor(observed, predicted, use = "complete.obs")^2,
    bias = mean(predicted - observed, na.rm = TRUE)
  )
}

Importance is not causation

Variable importance tells us which predictors helped this model make predictions, given this dataset and the other predictors in the model. It does not prove that a predictor causes methane flux to change.

Leave-one-site-out assessment

A random row split is useful, but it can be optimistic because rows from the same tower may appear in both training and testing. A harder question is:

Can the model predict methane flux at a tower site it has never seen?

Leave-one-site-out assessment fits one model per site. Each time, one site is held out for testing and all other sites are used for training.

To keep the workshop renderable, we use fewer trees for this assessment.

leave_one_site_out <- function(formula, data, ntree = 150, mtry = 2) {
  sites <- sort(unique(data$SITE))

  map_dfr(sites, function(site_id) {
    train_site <- data %>% filter(SITE != site_id)
    test_site <- data %>% filter(SITE == site_id)

    fit <- randomForest(formula,
                        data = train_site,
                        keep.forest = TRUE,
                        importance = FALSE,
                        mtry = mtry,
                        ntree = ntree)

    pred <- predict(fit, test_site)

    model_metrics(test_site$FCH4_F_gC, pred) %>%
      mutate(SITE = site_id,
             n_test = nrow(test_site))
  })
}

Two meanings of prediction

A random row split evaluates prediction for new observations from a familiar network of sites. Leave-one-site-out evaluates prediction for a new site. For final projects, be clear about which kind of prediction you want your model to support.

Select the best predictor from each category

For final model development, we will choose one strong predictor from each category:

  • climate: P_F, TA_F, or VPD_F;
  • vegetation: NDVI or EVI;
  • ecosystem: IGBP_model.

We will screen individual predictors using test RMSE. This is a simple strategy, not the only valid one.

screen_predictor <- function(predictor, category) {
  formula <- reformulate(predictor, response = "FCH4_F_gC")

  set.seed(111)
  fit <- randomForest(formula,
                      data = train,
                      keep.forest = TRUE,
                      importance = TRUE,
                      mtry = 1,
                      ntree = 300)

  pred <- predict(fit, test)

  model_metrics(test$FCH4_F_gC, pred) %>%
    mutate(category = category,
           predictor = predictor)
}

predictor_screen <- bind_rows(
  map_dfr(c("P_F", "TA_F", "VPD_F"), screen_predictor, category = "climate"),
  map_dfr(c("NDVI", "EVI"), screen_predictor, category = "vegetation"),
  screen_predictor("IGBP_model", category = "ecosystem")
)
