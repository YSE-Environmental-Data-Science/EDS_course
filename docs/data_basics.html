Data Basics

Reading the climate record of Yale-Myers Forest

Yale-Myers Forest is a 7,840 acre research forest in northeastern Connecticut. It is a hub for education, research, and harvesting operations within the Yale Forest School system. The forest primarily consists of mixed hardwoods on glacial till soils, with hemlock, scattered white pine stands from old field origins, and red pine plantations established in the 1940s.

In this workshop, we will use daily climate data to ask a more interesting question:

Has Yale-Myers Forest become warmer, wetter, or more climatically stressful since 1990?

By the end of the workshop, you will have built a small climate evidence brief for Yale-Myers Forest. Along the way, you will learn to:

  1. Download climate data with daymetr.
  2. Manipulate data with tidyverse:
    • Format date elements.
    • Use select() to choose variables from a data frame.
    • Use filter() to choose data based on values.
    • Use mutate() to create new variables.
    • Use reframe() to summarize datasets.
    • Use full_join() to merge datasets.
  3. Visualize data with ggplot2.
  4. Write basic functions that make your analysis easier to reuse.

Turn daily records into a usable climate timeline

Daymet stores time as year and day of year (yday). We need one actual date column before we can summarize the data by month, season, or year.

Step 1. Use the year and day of year to create a date:

yale_myers$year_day <- paste(yale_myers$year, yale_myers$yday, sep = "-") %>%
  as.Date(format = "%Y-%j")

Look at your work:

yale_myers$date <- format(yale_myers$year_day, "%Y-%m-%d") %>%
  as.Date()

Create climate variables with mutate()

The function mutate() creates new columns from existing columns. We will create several columns that help us interpret the climate record:

  • month: the calendar month.
  • month_name: the month name, used for plotting.
  • season: a rough ecological season.
  • julian_day: the day of year.
  • tmean: mean daily temperature.
  • hot_day: days with maximum temperature at or above 30 degrees C.
  • freeze_day: days with minimum temperature at or below 0 degrees C.
  • growing_season: May through September.
yale_myers <- yale_myers %>%
  mutate(month = format(date, format = "%m"),
         month_name = factor(month.abb[as.numeric(month)], levels = month.abb),
         julian_day = format(date, format = "%j"),
         season = case_when(
           month %in% c("12", "01", "02") ~ "Winter",
           month %in% c("03", "04", "05") ~ "Spring",
           month %in% c("06", "07", "08") ~ "Summer",
           month %in% c("09", "10", "11") ~ "Fall"
         ),
         tmean = (0.606 * as.numeric(tmax..deg.c.)) +
           (0.394 * as.numeric(tmin..deg.c.)),
         hot_day = tmax..deg.c. >= 30,
         freeze_day = tmin..deg.c. <= 0,
         growing_season = month %in% c("05", "06", "07", "08", "09"))

Introduction to ggplot2

ggplot2 is a plotting package in R. Graphics are built layer by layer. In this workshop, each plot should help answer part of the climate question instead of simply demonstrating a plotting command.

Load the library:

library(ggplot2)

Building blocks of the grammar of graphics

  • Data: the dataset.
  • Aesthetics: how variables map to visual features such as x-axis, y-axis, color, fill, size, labels, alpha, shape, line width, and line type.
  • Geometrics: how the data are displayed, such as points, lines, histograms, bars, or boxplots.
  • Facets: split the data into subsets.
  • Statistics: binning, smoothing, summaries, or model fits.
  • Coordinates: the space between data and display.
  • Themes: non-data visual elements.

Data layer

We first define the source of the information to be visualized:

ggplot(data = yale_myers_annual) +
  labs(title = "Annual climate record of Yale-Myers Forest")

Aesthetic layer

The aesthetic layer maps variables to visual features. Here, year is mapped to the x-axis and annual mean temperature is mapped to the y-axis:

ggplot(data = yale_myers_annual,
       aes(x = year, y = tmean)) +
  labs(title = "Annual mean temperature at Yale-Myers Forest")

Geometric layer

The geometric layer controls how the data are drawn:

ggplot(data = yale_myers_annual,
       aes(x = year, y = tmean)) +
  geom_point() +
  geom_line() +
  labs(title = "Annual mean temperature at Yale-Myers Forest",
       x = "Year",
       y = "Mean air temperature (degrees C)")

Add size and color to show whether warmer years also had more hot days:

ggplot(data = yale_myers_annual,
       aes(x = year, y = tmean, size = hot_days, color = hot_days)) +
  geom_point() +
  labs(title = "Warm years and hot days at Yale-Myers Forest",
       x = "Year",
       y = "Mean air temperature (degrees C)",
       size = "Hot days",
       color = "Hot days")

Create a histogram to see the distribution of daily mean temperature:

ggplot(data = yale_myers_sub,
       aes(x = tmean)) +
  geom_histogram(binwidth = 5) +
  labs(title = "Distribution of daily mean temperature",
       x = "Daily mean air temperature (degrees C)",

Facet layer

The facet layer splits the data into subsets. Here we compare fall months, which are often important for field work and forest management:

yale_myers_fall <- yale_myers_sub %>%
  filter(month > "08" & month < "12")

p <- ggplot(data = yale_myers_fall,
            aes(x = date, y = tmean)) +
  geom_point()

p + facet_grid(month_name ~ .) +
  labs(title = "Fall temperature records at Yale-Myers Forest",
       x = "Date",
       y = "Mean air temperature (degrees C)")

Statistics layer

We can add a model fit to ask whether a seasonal climate variable is changing over time. Here we explore February temperatures from 1990-2018:

yale_myers_feb <- yale_myers_sub %>%
  filter(month == "02")

ggplot(data = yale_myers_feb,
       aes(x = date, y = tmean)) +
  geom_point() +
  stat_smooth(method = lm, col = "red") +
  labs(title = "Are February temperatures changing at Yale-Myers Forest?",
       x = "Date",
       y = "Mean air temperature (degrees C)")
`geom_smooth()` using formula = 'y ~ x'

Comparing Yale-Myers Forest to New Haven

Yale-Myers Forest is inland and forested. New Haven is coastal and urban. Comparing the two locations gives full_join() a purpose: we can ask whether the same years were warm at both locations.

Download Daymet data for New Haven:

new_haven_daymet <- download_daymet(lat = 41.3083,
                                     lon = -72.9279,
                                     start = 1990,
                                     end = 2018)

Done !
new_haven <- as.data.frame(new_haven_daymet$data)

Prepare the New Haven data using the same steps:

new_haven_sub <- new_haven %>%
  mutate(date = paste(year, yday, sep = "-") %>%
           as.Date(format = "%Y-%j"),
         month = format(date, format = "%m"),
         tmean = (0.606 * as.numeric(tmax..deg.c.)) +
           (0.394 * as.numeric(tmin..deg.c.))) %>%
  rename(tmax = tmax..deg.c.,
         tmin = tmin..deg.c.,
         prcp = prcp..mm.day.) %>%
  select(date, month, year, tmax, tmin, tmean, prcp)

Create a mean annual temperature summary for New Haven:

new_haven_annual_tmean <- new_haven_sub %>%
  reframe(.by = year,
          new_haven = mean(tmean, na.rm = TRUE))

Use full_join() to combine the annual temperature summaries:

annual_tmean_compare <- yale_myers_annual_tmean %>%

Introduction to writing functions in R

Functions are reusable sets of instructions. They help you avoid repeating code and make your analysis easier to check. In environmental data science, functions are especially useful when you need to apply the same summary to multiple locations, variables, or time periods.

What is a function?

In R, a function is an object that performs a task. R comes with built-in functions, such as:

  • min(), max(), mean(), and median().
  • sum().
  • range().
  • str().
  • print().
  • length().
  • sort().

You can also create your own functions. The basic syntax is:

function_name <- function(parameters) {
  # code to run goes here
}

Write a standard error function

The standard error measures the precision of an estimated mean. We will use this function in the assignment:

standard_error <- function(x) {
  sd(x, na.rm = TRUE) / sqrt(sum(!is.na(x)))
}

Test the function:

standard_error(yale_myers_sub$tmean)
[1] 0.09504815

Write a reusable monthly climate summary

Now write a function that summarizes monthly climate conditions for any Daymet-style data frame that already has month, tmean, prcp, tmin, and tmax columns:

summarize_monthly_climate <- function(data) {
  data %>%
    reframe(.by = month,
            tmean = mean(tmean, na.rm = TRUE),
            tmean_se = standard_error(tmean),
            prcp = mean(prcp, na.rm = TRUE),
            prcp_se = standard_error(prcp),
            tmin = mean(tmin, na.rm = TRUE),
            tmin_se = standard_error(tmin),
            tmax = mean(tmax, na.rm = TRUE),
            tmax_se = standard_error(tmax)) %>%
    arrange(month)
}

Use the function on both locations:

yale_myers_monthly_summary <- summarize_monthly_climate(yale_myers_sub)
new_haven_monthly_summary <- summarize_monthly_climate(new_haven_sub)

Add location labels and combine the data:

monthly_location_compare <- bind_rows(
  yale_myers_monthly_summary %>% mutate(location = "Yale-Myers Forest"),
  new_haven_monthly_summary %>% mutate(location = "New Haven")
)

Plot monthly mean temperature with standard error:

ggplot(data = monthly_location_compare,
       aes(x = month, y = tmean, color = location, group = location)) +
  geom_line() +
  geom_point() +
  geom_errorbar(aes(ymin = tmean - tmean_se,
                    ymax = tmean + tmean_se),
                width = 0.2) +
  labs(title = "Monthly mean temperature with standard error",
       x = "Month",
       y = "Mean air temperature (degrees C)",
       color = "Location")

Function arguments by position and by name

Function arguments can be passed by position:

subtract_two_nums <- function(x, y) {
  x - y
}

print(subtract_two_nums(3, 1))
[1] 2

Or by name:

print(subtract_two_nums(x = 3, y = 1))
[1] 2
print(subtract_two_nums(y = 1, x = 3))
[1] 2

When functions become more complex, named arguments often make your code easier to read.
