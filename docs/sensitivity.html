Sensitivity Analysis: Stress Testing a Methane Flux Model

From prediction to model trust

In the Random Forest workshop, we built a model to predict monthly methane flux. A prediction is only useful if we understand how the model responds to changing inputs and where the model is being asked to extrapolate beyond the data it learned from.

In this workshop, we will ask:

What does our methane model believe about climate, vegetation, and ecosystem controls, and can we trust those beliefs for Yale-Myers-like wetland conditions?

By the end of this workshop, you will be able to:

  1. Identify which predictors are used by a fitted random forest model.
  2. Summarize the training conditions used to fit the model.
  3. Build one-at-a-time sensitivity curves.
  4. Compare realistic methane flux scenarios.
  5. Flag scenarios that fall outside the model’s training range.
  6. Build two-way sensitivity heatmaps to explore interactions.
  7. Compare sensitivity results with random forest variable importance.
  8. Write a short model-risk interpretation for final project reporting.

A reference condition

We need a baseline row that contains one value for every predictor in the model. Numeric predictors are set to their median training value. Categorical predictors are set to their most common training level.

mode_value <- function(x) {
  tab <- sort(table(x), decreasing = TRUE)
  names(tab)[1]
}

make_reference_row <- function(data, predictors) {
  reference <- map(predictors, function(predictor) {
    x <- data[[predictor]]
    if (is.numeric(x)) {
      median(x, na.rm = TRUE)
    } else {
      mode_value(x)
    }
  })

  names(reference) <- predictors
  as_tibble(reference)
}

reference_row <- make_reference_row(train_data, final_predictors)

reference_row <- reference_row %>%
  mutate(across(all_of(categorical_predictors), ~ factor(.x, levels = levels(train_data[[cur_column()]]))))

One-at-a-time sensitivity

One-at-a-time sensitivity varies one predictor across its observed training range while holding the other predictors constant. This is a simple way to ask how strongly the model responds to each variable.

make_oat_grid <- function(predictor, data, reference, n = 60) {
  x <- data[[predictor]]

  if (!is.numeric(x)) {
    return(NULL)
  }

  varied_values <- tibble(
    !!predictor := seq(min(x, na.rm = TRUE),
                       max(x, na.rm = TRUE),
                       length.out = n)
  )

  reference %>%
    select(-all_of(predictor)) %>%
    slice(rep(1, n)) %>%
    bind_cols(varied_values)
}

oat_predictions <- map_dfr(numeric_predictors, function(predictor) {
  grid <- make_oat_grid(predictor, train_data, reference_row)
  grid$predicted_flux <- predict(fch4_rf, newdata = grid)
  grid$varied_predictor <- predictor
  grid$varied_value <- grid[[predictor]]
  grid
})

Yale-Myers-style scenarios

Scenario analysis asks how the model responds to meaningful combinations of conditions. The exact Yale-Myers methane flux is unknown here, so these are modeling scenarios, not field observations.

q_value <- function(predictor, probability) {
  quantile(train_data[[predictor]], probability, na.rm = TRUE)
}

set_if_present <- function(data, predictor, value) {
  if (predictor %in% names(data)) {
    data[[predictor]] <- value
  }
  data
}

scenario_row <- function(name, temperature_q, precipitation_q, vpd_q = 0.5,
                         ndvi_q = 0.5, evi_q = 0.5) {
  row <- reference_row %>%
    select(all_of(final_predictors))

  row <- set_if_present(row, "TA_F", q_value("TA_F", temperature_q))
  row <- set_if_present(row, "P_F", q_value("P_F", precipitation_q))
  row <- set_if_present(row, "VPD_F", q_value("VPD_F", vpd_q))
  row <- set_if_present(row, "NDVI", q_value("NDVI", ndvi_q))
  row <- set_if_present(row, "EVI", q_value("EVI", evi_q))

  row %>%
    mutate(scenario = name, .before = 1)
}

scenario_table <- bind_rows(

Training-range flags

A scenario can look precise while still being outside the model’s experience. This helper checks whether scenario values are inside the numeric range of the training data and whether categorical levels were observed during training.

flag_training_range <- function(scenarios, training_data, predictors) {
  numeric_flags <- map_dfr(intersect(predictors, numeric_predictors), function(predictor) {
    min_value <- min(training_data[[predictor]], na.rm = TRUE)
    max_value <- max(training_data[[predictor]], na.rm = TRUE)

    scenarios %>%
      transmute(
        scenario,
        predictor = predictor,
        value = as.character(.data[[predictor]]),
        inside_training_domain = .data[[predictor]] >= min_value & .data[[predictor]] <= max_value
      )
  })

  categorical_flags <- map_dfr(intersect(predictors, categorical_predictors), function(predictor) {
    observed_levels <- unique(as.character(training_data[[predictor]]))

    scenarios %>%
      transmute(
        scenario,
        predictor = predictor,
        value = as.character(.data[[predictor]]),
        inside_training_domain = as.character(.data[[predictor]]) %in% observed_levels
      )
  })

  bind_rows(numeric_flags, categorical_flags)
}

Two-way sensitivity

Methane flux can respond to combinations of conditions. Two-way sensitivity varies two predictors at the same time while holding the others at their reference values.

make_two_way_grid <- function(x_predictor, y_predictor, data, reference, n = 45) {
  x_values <- seq(min(data[[x_predictor]], na.rm = TRUE),
                  max(data[[x_predictor]], na.rm = TRUE),
                  length.out = n)

  y_values <- seq(min(data[[y_predictor]], na.rm = TRUE),
                  max(data[[y_predictor]], na.rm = TRUE),
                  length.out = n)

  varied_grid <- crossing(
    !!x_predictor := x_values,
    !!y_predictor := y_values
  )

  reference %>%
    select(-all_of(c(x_predictor, y_predictor))) %>%
    slice(rep(1, nrow(varied_grid))) %>%
    bind_cols(varied_grid)
}

two_way_pairs <- list(
  c("TA_F", "P_F"),
  c("TA_F", "VPD_F"),
  c("TA_F", "NDVI")
) %>%
  keep(~ all(.x %in% numeric_predictors))

two_way_predictions <- imap_dfr(two_way_pairs, function(pair, pair_id) {
  grid <- make_two_way_grid(pair[1], pair[2], train_data, reference_row)
  grid$predicted_flux <- predict(fch4_rf, newdata = grid)
  grid$x_predictor <- pair[1]
  grid$y_predictor <- pair[2]
  grid$x_value <- grid[[pair[1]]]
  grid$y_value <- grid[[pair[2]]]
  grid$pair <- paste(pair, collapse = " x ")
  grid
})

Compare sensitivity with importance

Variable importance ranks predictors by how much they contributed to model performance. Sensitivity curves ask how much predictions change when a predictor is varied across its training range. They are related, but they are not the same.

Model-risk memo

Sensitivity analysis is not just a plot-making step. It is part of model interpretation. For Yale-Myers wetlands, we especially care about whether high-methane scenarios are supported by the training data.

Use the objects created above to answer:

  1. Which predictor produced the largest one-at-a-time change in predicted methane flux?
  2. Which scenario produced the highest predicted methane flux?
  3. Did the Yale-Myers-like wetland scenario stay inside the model’s training domain?
  4. Do variable importance and sensitivity tell the same story?
  5. What would make you cautious about using this model to estimate methane flux for Yale-Myers wetlands?
