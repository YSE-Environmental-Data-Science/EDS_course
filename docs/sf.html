Change the CRS to a projected EPSG with st_transform.

Coordinate reference systems (CRS) are like measurement units for coordinates: they specify which location on Earth a particular coordinate pair refers to. Simple features have two attributes to store a CRS: epsg and proj4string. This implies that all geometries in a geometry list-column must have the same CRS. Both may be NA, e.g. in case the CRS is unknown, or when we work with local coordinate systems (e.g. inside a building, a body, or an abstract space).

proj4string is a generic, string-based description of a CRS, understood by the PROJ library. It defines projection types and (often) defines parameter values for particular projections, and hence can cover an infinite amount of different projections. This library (also used by GDAL) provides functions to convert or transform between different CRS. epsg is the integer ID for a particular, known CRS that can be resolved into a proj4string. Some proj4string values can be resolved back into their corresponding epsg ID, but this does not always work.

The importance of having epsg values stored with data besides proj4string values is that the epsg refers to a particular, well-known CRS, whose parameters may change (improve) over time; fixing only the proj4string may remove the possibility to benefit from such improvements, and limit some of the provenance of datasets, but may help reproducibility. Coordinate reference system transformations can be carried out using st_transform.

fluxnet_ch4_sf <- st_transform(fluxnet_ch4_sf, crs = 4087)

Getting country and state boundaries with rnaturalearth

rnaturalearth provides world map data at several resolutions, including country boundaries with a continent attribute you can filter or group by. Its companion package, rnaturalearthhires, adds higher-resolution boundaries down to the state/province level for individual countries. Together they’re a good general-purpose source for the kind of reference boundaries you’ll often need to visualize or subset other spatial data against.

Key functions:

ne_countries(): Returns country boundaries as an sf object, with a continent column you can filter or group by.

ne_states(): Returns state/province boundaries for a given country (from rnaturalearthhires).

Example:

library(rnaturalearth)
library(rnaturalearthhires)

world <- ne_countries(scale = "medium", returnclass = "sf")

# Get New York state from the US
us_states <- ne_states(country = "United States of America", returnclass = "sf")
new_york <- us_states %>% filter(name == "New York")
new_york %>% ggplot() + geom_sf()

Get a simple feature of every country in South America, and one that unions them into a single South America polygon:

south_america_countries <- world %>% filter(continent == "South America")
south_america <- south_america_countries %>% st_union() %>% st_as_sf()
south_america %>% ggplot() + geom_sf()

A straight st_intersection() can return more or fewer rows than you started with (a point can sit exactly on a shared border and match two polygons, or fall outside all of them, e.g. small islands at this map resolution), so join back on SITE_ID instead of assigning the result directly:

country_lookup <- st_intersection(aoi_terrestrial, fluxnet_ch4_sf) %>%
  st_drop_geometry() %>%
  select(SITE_ID, name) %>%
  distinct(SITE_ID, .keep_all = TRUE)

fluxnet_ch4_sf <- fluxnet_ch4_sf %>%
  left_join(country_lookup, by = "SITE_ID") %>%
  rename(country = name)

Writing files using st_write:

When writing, you can use the following arguments to control update and delete: update=TRUE causes an existing data source to be updated, if it exists; this option is by default TRUE for all database drivers, where the database is updated by adding a table.

delete_layer=TRUE causes st_write to try to open the data source and delete the layer; no errors are given if the data source is not present, or the layer does not exist in the data source.

delete_dsn=TRUE causes st_write to delete the data source when present, before writing the layer in a newly created data source. No error is given when the data source does not exist. This option should be handled with care, as it may wipe complete directories or databases.
