Vector Data: Mapping Gaps in the Global Methane Tower Network

Mapping methane observations

Models are only as good as the places where we collect data. FLUXNET towers provide direct measurements of ecosystem-atmosphere exchange, and FLUXNET-CH4 towers are especially important for understanding methane emissions from natural ecosystems. But the tower network is not evenly distributed across the planet.

In this workshop, we will use vector data to ask:

If FLUXNET CH4 towers are used to understand natural methane emissions, where are we observing the planet, and where are we guessing?

By the end of the workshop, you will be able to:

  1. Convert tabular coordinates into simple feature objects.
  2. Inspect geometries and coordinate reference systems.
  3. Transform vector data into an appropriate projected CRS.
  4. Measure distances among point locations.
  5. Use country and continent polygons from rnaturalearth.
  6. Use spatial predicates and spatial joins to count towers inside regions.
  7. Calculate polygon area and tower density.
  8. Evaluate basic representativeness of the CH4 tower network.
  9. Write spatial outputs for later workshops.

From tables to simple features

We are interested in the sites with methane data. The column FLUXNET-CH4 is read into R as FLUXNET.CH4, because R replaces the hyphen with a dot.

fluxnet_ch4 <- fluxnet_sites %>%
  filter(FLUXNET.CH4 != "")

Latitude and longitude become spatial data when we tell R which columns contain coordinates and what coordinate reference system they use. A few rows in the full FLUXNET table may be missing coordinates, so remove those before converting the table to points. These coordinates are in WGS84 longitude/latitude, EPSG 4326.

fluxnet_sites_located <- fluxnet_sites %>%
  filter(!is.na(LOCATION_LONG), !is.na(LOCATION_LAT))

fluxnet_ch4_located <- fluxnet_ch4 %>%
  filter(!is.na(LOCATION_LONG), !is.na(LOCATION_LAT))
fluxnet_sites_sf <- st_as_sf(x = fluxnet_sites_located,
                              coords = c("LOCATION_LONG", "LOCATION_LAT"),
                              crs = 4326)

fluxnet_ch4_sf <- st_as_sf(x = fluxnet_ch4_located,
                            coords = c("LOCATION_LONG", "LOCATION_LAT"),
                            crs = 4326)

Coordinate reference systems (CRS) define how coordinates map to locations on Earth. Longitude/latitude coordinates are useful for storing global locations, but projected coordinates are better when measuring area and distance. We will transform the tower data to World Equidistant Cylindrical, EPSG 4087, so distance and area operations use meters.

fluxnet_sites_sf <- st_transform(fluxnet_sites_sf, crs = 4087)
fluxnet_ch4_sf <- st_transform(fluxnet_ch4_sf, crs = 4087)

Are methane towers clustered?

If methane towers are clustered, models may describe those places well while leaving other climate or ecosystem conditions underrepresented. Use st_distance() to calculate the distance among CH4 towers, then identify each tower’s nearest neighbor.

distance_matrix_km <- as.matrix(st_distance(fluxnet_ch4_sf)) / 1000
diag(distance_matrix_km) <- NA

fluxnet_ch4_sf$nearest_neighbor_km <- apply(distance_matrix_km, 1, min, na.rm = TRUE)

Map nearest-neighbor distance. Larger values identify more isolated towers:

ggplot(data = fluxnet_ch4_sf) +
  geom_sf(aes(color = nearest_neighbor_km), size = 2) +
  labs(title = "Nearest CH4 tower distance",
       color = "Nearest tower (km)")

Look at the distribution:

ggplot(data = fluxnet_ch4_sf,
       aes(x = nearest_neighbor_km)) +
  geom_histogram(color = "black", fill = "white") +
  labs(title = "Distribution of nearest-neighbor distances",
       x = "Distance to nearest CH4 tower (km)",
       y = "Count")

Add country and continent context

Point locations become more useful when we attach contextual information. We will get country polygons from rnaturalearth and spatially join each tower to a country and continent.

world <- ne_countries(scale = "medium", returnclass = "sf") %>%
  filter(continent %in% c("Europe", "Asia", "North America",
                          "South America", "Africa", "Oceania")) %>%
  st_transform(crs = 4087) %>%
  st_make_valid()

Map the global land polygons:

ggplot(data = world) +
  geom_sf(fill = "gray95", color = "gray70") +
  labs(title = "Country polygons from rnaturalearth")

Attach country and continent to all FLUXNET sites and to CH4 sites:

world_context <- world %>%
  select(country = name, continent, geometry)

fluxnet_sites_sf <- fluxnet_sites_sf %>%
  st_join(world_context, join = st_within)

fluxnet_ch4_sf <- fluxnet_ch4_sf %>%
  st_join(world_context, join = st_within)

Case study: South America and Brazil

Tropical and subtropical wetlands are important to the global methane cycle. South America is a useful case study because it includes large wetland systems and major climate gradients.

Create a simple feature object for South America, and a second object that unions all South American countries into one polygon:

south_america_countries <- world %>%
  filter(continent == "South America")

south_america <- south_america_countries %>%
  st_union() %>%
  st_as_sf() %>%
  rename(geometry = x)

ggplot(data = south_america) +
  geom_sf() +
  labs(title = "South America as one polygon")

Basic representativeness analysis

Now we can ask whether FLUXNET CH4 sites are representative of the larger FLUXNET network. This is not a complete representativeness analysis, but it is a useful first pass. We will compare:

  • geographic representation by continent;
  • ecosystem representation by IGBP;
  • climate space coverage using mean annual temperature (MAT) and precipitation (MAP);
  • elevation coverage.

Representation by continent

Count all FLUXNET sites by continent and CH4 sites by continent:

all_sites_by_continent <- fluxnet_sites_sf %>%
  st_drop_geometry() %>%
  count(continent, name = "all_fluxnet_sites")

ch4_sites_by_continent <- fluxnet_ch4_sf %>%
  st_drop_geometry() %>%
  count(continent, name = "ch4_sites")

Join the summaries and calculate the fraction of FLUXNET sites that have CH4 data:

representation_by_continent <- all_sites_by_continent %>%
  full_join(ch4_sites_by_continent, by = "continent") %>%
  mutate(ch4_sites = replace_na(ch4_sites, 0),
         ch4_fraction = ch4_sites / all_fluxnet_sites) %>%
  arrange(desc(ch4_fraction))

Writing spatial files

We will use the CH4 tower simple feature in later workshops. Save a shapefile version:

write_sf(fluxnet_ch4_sf, "data/products/fluxnet_ch4.shp")

It is also possible to convert a simple feature object back to a data frame:

fluxnet_ch4_df <- as.data.frame(fluxnet_ch4_sf)

Check the class:

class(fluxnet_ch4_df)
[1] "data.frame"

Once the object is a data frame, it no longer has dedicated sf methods. The geometry column is still present, but R no longer treats the object as a spatial layer. Write a CSV version:

write.csv(fluxnet_ch4_df, "data/fluxnet_ch4.csv")
